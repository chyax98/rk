# RenderKit 1.0 中文优先写作规则

状态：已采纳  
日期：2026-05-17

## 背景

用户明确指出：这是一个中文区项目，不要写太多英文。

因此 RenderKit 后续文档、过程记录、研究报告、产品说明默认采用中文。英文只保留在必要位置，例如代码路径、命令、API 名称、库名、专有名词和原始引用。

## 规则

1. **默认中文。** 新增产品文档、研究文档、过程文档优先用中文写。
2. **先中文结论，再证据。** 保持“结论 → 依据 → 下一步”的结构。
3. **英文只用于不可避免的技术标识。** 例如：`pnpm verify`、`ArtifactView.jsx`、`TextQuoteSelector`、`SQLite`、`RenderKit`。
4. **外部 repo 分析也要中文化。** 可以保留 repo 名、文件路径、原文术语，但分析和建议用中文表达。
5. **面向 review 的文档要可读。** 避免大段英文叙述堆砌；必要时增加中文摘要、优先级表和行动清单。
6. **Agent authoring skill 可以中英混合，但中文优先。** 因为 `.rk.md` 可能服务中文读者，写作约束要提醒 Agent 处理中英混排、中文排版和盘古之白。
7. **已有英文文档不强制一次性全改。** 后续触碰相关文档时逐步补中文摘要或改写，避免无意义重写打断产品建设。

## 后续执行

从下一批文档开始：

- `docs/product/*` 默认中文；
- `research/design-assets/*` 默认中文；
- pass 文档默认中文；
- final audit 默认中文；
- 代码注释可按上下文中英混合，但面向用户/评审的解释用中文。

## 验收标准

后续 review 时，如果一个面向用户或过程记录的文档英文占比过高，需要补充中文摘要或改写关键章节。
